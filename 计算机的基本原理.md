## 一、计算机的基本原理
  第一次数学危机 (无理数)  
    希帕索斯悖论  
  第二次数学危机 (微积分)  
    贝克莱悖论  
  第三次数学危机 (康托尔创立集合论)  
    罗素悖论:S由一切不是自身元素的集合所组成,S是否属于S?  
    哥德尔不完备性定理:既没有办法证明，又没有办法推翻  
  图灵机 (定义可计算性)  
    组成:1.一条存储带:双向无限长,有小方格,小方格内可存储一个数字  
         2.一个控制器:存储当前自身的状态  
         3.一个读写头,可以从纸带上读/写数据,向左/右移动  
    图灵可计算:  
      如果一个函数f存在一台图灵机，对于任何输入w，该图灵机都能在纸带上输出f(w)然后停机，则称f是图灵可计算的。  
    通用图灵机:存在一台特殊的图灵机，可以模拟其他图灵机的行为  
    任何在算法上可计算的问题，都可以被一台图灵机计算  
    停机问题：  
      不存在一台图灵机能够正确判定所有(M,w)对的停机问题  
    平衡计算量和存储量:e  
      如果使用高进制，存储的程序会变长  
      如果使用低进制：计算时移动次数变多  
    已解决问题:二进制表示  
    待解问题:如何进行计算 加法:本位xor 进位and   

    与非门/或非门的电路实现  
  第一代计算机:20世纪40-50年代  

## 二、计算机的硬件系统  
  ### 1.从外观出发：主机、显示屏、键盘……  
    输入设备、主机、基本输出设备、其他外部设备、通讯设备  
    专门有风扇贴在CPU表面散热  
    主板:通过总线(数据、地址、控制)将CPU、存储器、各种输入/输出设备连接  
         芯片组(南桥、北桥):控制链接  
         提供各种接口
  ### 2.冯诺依曼结构  
    输入设备、输出设备、存储器(把数据信号传输给其他四个部件)、运算器(与存储器的数据传输双向)、控制器(控制其他四个部件工作)  
    (1)中央处理器CPU(微处理器)  
      控制器  
      运算器(ALU):算数运算+逻辑运算; 寄存器(暂时存储等待处理的数据)+累加器  

      字长:CPU一次处理的二进制位数  
        越长则处理速度越快，精度越高(32/64位)  

      主频率:时钟频率(工作频率)Hz  
        决定计算机的运行速度(1.6GHz/3.4GHz) 越高越快  

      计算机能执行哪些指令由硬件决定，并决定了指令的机构  
      高级语言是硬件无关的，而低级语言是硬件有关的  

      指令集合:CPU所能够处理的全部指令的集合，CPU的根本属性，不同结构的计算机有不同的指令集合  
      CISC (Complex)复杂指令集计算机:大量指令以及相应的硬件逻辑  
        昂贵、指令集大、电路复杂、功耗大(eg超算)  
      RISC (Reduced)精简指令集计算机:只包含最基本功能，其他复杂指令由简单指令组合  
        便宜、指令集小、电路简单、功耗低(eg手机) 使用越来越多  
      X86 Intel&AMD 的CISC指令系统  
      ARM、POWER(IBM)、APPLE M1 RISC指令系统  

      多核处理器:  
        好处:通过划分任务，线程应用能够充分利用多个执行内核;主频不变，特定时间内执行更多任务  
        *程序:以某些程序设计语言编写的指令序列  
         进程:一个应用程序在计算机上的一次执行过程  
         线程:进程的一部分，一个进程包含多个线程  

        GPU:40+%是ALU 专门处理并行操作  
      
      分布式计算  

      CPU寻址空间  
        32位CPU的物理地址 2^32=4GB个地址  

    (2)内部存储器  
      存储容量单位:位(bit)表示数据最小单位;字节(Byte)存储信息最小单位  
      CPU寄存器  
      ROM只读存储器(read only memory):断电后信息不消失  
        主要存储BIOS(Basic Input Output System)  
        在出厂时就将中断服务程序、启动程序固化在其中，称为固件  
        *操作系统存储在外部存储器中  
      RAM随机存储器:断电后信息消失  
        内存一般指RAM  
        静态随机存储器(SRAM)和动态随机存储器(DRAM)  
        SRAM:不需要刷新。成本高，功耗低，速度快  
        DRAM:需要定时刷新保存数据。成本低，功耗高，速度慢  
        DDR SDRAM:(double data rate)双倍速率同步动态随机存储器:一个时钟周期传输两次数据  
      Cache高速缓冲存储器  
        内存存取速度往往比CPU慢，在CPU和内存中引入Cache(与CPU速度接近，容量较小)  
        程序的局部性原理:  
          时间局部性:如果一个地址正在被使用，那么它最近很有可能再次被使用  
          空间局部性:如果一个地址正在被使用，那么它附近的空间在最近可能被使用  
        缓存命中率:CPU需要访问的数据恰好在缓存中的概率  
        替换算法(最大化未来的缓存命中率)  
          FIFO 先进先出  
          LFU 最不经常使用  
          LRU 最近最少使用  
      存储器的指标:容量和访问速度  
        一般个人电脑8~32G  
        访问速度1800MHz、2500MHz(最近DDR5速度和CPU速度相当甚至更快，但总线延时还是慢)  
        容量影响处理速度(特别是大型文件)  
    (3)外部存储器(主板之外)  
      容量大、存取速度低  
      主要种类:硬盘、光盘、软盘、磁盘  
      具有非易失性  
      机械硬盘(HDD) 速度慢、体积大;成本低  
      固态硬盘(SSD) 速度快、体积小;成本高  

      *虚拟内存(如果程序太大，超过内存)  
        当程序启动时，在硬盘上虚拟出一片"内存"  
      云存储  

    (4)输入/输出设备  
      早期计算机采用等待方式向外部设备发出请求，在外部设备准备期间，一直等待  
      中断方式向外部设备发出请求，在外部设备准备期间，干别的事情  
      DMA(direct memory access)方式把数据发给一个直接访问控制器，解放CPU，通过硬件线路直接实现  

    (5)南北桥芯片  
      北桥芯片:在主板芯片组起主导作用(现在)  
      南桥芯片:负责I/O总线的通信 如USB、LAN etc.  

    (6)总线BUS  
      数据总线:  
      地址总线:  
      控制总线:控制器单向传出、输入/出设备双向控制  
      总线宽度=字长  
      总线标准: ISA,PCI,AGP,PCI-E,USB,USB-C  
    (7)外围设备 插到适配器上  
## 三、软件基础  
  应用含义:指示计算机完成任务的指令序列和相关数据  
  学科含义:软件=程序+文档  
  传统软件以拷贝的产品形式存在  
  现在以云模式使用  
  ### 1.分类  
    系统软件:操作系统:Windows,MacOS,iOs  
             程序语言环境 C,Basic,汇编  
             数据库:Access,IBMDB2,Oracle  
    应用软件  

  ### 2.操作系统:使计算机从专用到通用  
    管理硬件资源控制程序运行，改善人机界面，为应用软件提供支持(shell)  
    使用者视角:一台虚拟机，对底层资源细节的抽象，提供更方便的人机界面  
    发展历程：分布式操作系统、网络中间件、局域网操作系统、云/数据中心操作系统  
    UNIX 现代操作系统的代表:C语言编写 可移植  
    DOS (Disk Operation System) IBM  
    Windows 3.1 运行于DOS之上;Windows95为较为完整的操作系统  

    其他巨头——本质上也是以软件服务为核心eg:NVIDIA CUDA  

    Linux 著名微机开源操作系统:减少重复开发  

    为什么要操作系统  
      五大功能:进程管理、存储管理、设备管理、文件管理、用户界面  
      <1>进程管理:一个cpu同一时刻只能处理一个进程  
        顺序运行->并发运行  
        进程:使程序能够并发运行  
          程序与进程的区别:进程更能描述并发;进程有生命周期,动态;进程包含程序和数据  
          
      <2>存储管理  
        帕金森定律:内存多大、程序多长  
        andy bill rule:硬件跑得越快、软件越来越复杂,使得运行速度没有明显增快  
        固定分区:分区大小相等，设置内存分配表，内存利用率不高  
        可变分区:分区大小不等，分配时按需分割内存  
      
      <3>设备管理  
        中断管理  

      <4>文件管理  
        存储、检索信息  
        树  

      <5>用户界面-虚拟/增强现实
